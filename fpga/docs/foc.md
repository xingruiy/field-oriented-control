# Field-Oriented Control — Theory, Math and Implementation

The FOC current/torque inner loop: `rtl/foc/` (`foc_pkg`, `sincos_lut`,
`clarke`, `park`, `inv_park`, `pi_controller`, `svpwm`, `foc_core`) and
how the pieces compose. Companion documents: `docs/pwm.md` (gate
generation, sampling instant), `docs/hall.md` (angle source),
`docs/config.md` (operating point) and `docs/hardware.md` (datasheet
rationale).

---

## 1. Why field-oriented control

A 3-phase PM machine produces torque $T = K_t \, i_q$ where $i_q$ is the
stator current component *in quadrature with* the rotor flux. Current
along the flux axis ($i_d$) produces no torque, only losses (and field
weakening, a non-goal here). FOC rotates the measured 3-phase currents
into the rotor (d/q) reference frame, where:

- the sinusoidal steady-state currents become **DC quantities**, so plain
  PI controllers achieve zero steady-state error;
- torque control reduces to regulating $i_q$ to a reference with
  $i_d \rightarrow 0$.

In the rotor frame the machine's electrical dynamics are

$$v_d = R i_d + L \frac{di_d}{dt} - \omega_e L i_q, \qquad
  v_q = R i_q + L \frac{di_q}{dt} + \omega_e L i_d + \omega_e K_e$$

For this motor (**per-phase** $R_s$ = 1.58 Ω, $L_s$ = 127 µH — the
datasheet's 3.16 Ω / 253 µH are phase-to-phase values; the dq plant uses
per-phase, as the working STM32 reference does) each axis is a
first-order RL plant with $\tau_e = L_s/R_s \approx 80\ \mu s$. The
cross-coupling and back-EMF terms ($\omega_e L i$, $\omega_e K_e$) are
treated as disturbances and left to the integrators (no feed-forward
decoupling in v1 — at ≤ 257 Hz electrical they are slow compared to the
1 kHz loop).

## 2. Number formats (`foc_pkg.sv`)

| Type | Format | Used for |
|---|---|---|
| `q15_t` | Q1.15, ±1.0 | all module I/O: currents, voltages, duties, sin/cos |
| `q13_t` | Q3.13, ±4.0 | headroom format where √3-scale intermediates appear |
| `angle_t` | unsigned 16 bit | electrical angle, full circle = $2^{16}$ codes (wrap = free modulo) |
| PI gains | Q4.12, ±8 | `kp`, `ki` |
| PI integrator | Q5.27 in 32 bit | clamped to ±1.0 |

Scaling anchors (locked):

- **current**: q15 1.0 = 1.25 A (CSA gain 1.2 V/A → ±1.25 A full scale);
- **voltage**: q15 1.0 = $V_{bus}$ = 24 V (duties are normalized to the
  DC link; measured vbus is telemetry-only in v1).

Conventions enforced everywhere: multiplies produce full-width products;
rounding is **round-half-up** (`rnd_shr`: add $2^{n-1}$, arithmetic shift
right $n$); saturation (`sat16`) only at format boundaries, with a `sat`
flag exported so telemetry can see any clipping (`sat_any`).

Every multiply has its own pipeline register — a Spartan-7 DSP48 takes
≈ 6–7 ns combinationally, so unregistered products dominated the failed
100 MHz timing runs. Each math block is therefore a small valid-piped
pipeline (latencies in §8).

## 3. Angle → sin/cos (`sincos_lut.sv`)

Quarter-wave LUT with linear interpolation, one RAMB36, 3-clk latency:

- quadrant folding: `quad = θ[15:14]`, phase `p = θ[13:0]`; odd quadrants
  mirror ($16384 - p$), quadrant MSB supplies the sign — the table only
  stores the first quadrant;
- 1024 entries of `{value Q1.17, slope}`; output
  $y = val + slope \cdot frac/16$ with `frac = c[3:0]`;
- $\cos\theta = \sin(\theta + 0x4000)$ — the second BRAM port serves cos,
  so one BRAM yields both;
- final rounding Q1.17 → Q1.15; total error < 1 LSB of Q1.15 against
  double-precision sin (exhaustively checked over all $2^{16}$ angles in
  `tb_sincos_lut`).

Table generated by `scripts/gen_sincos_lut.py` (checked in as
`rtl/math/sincos_lut.mem`).

## 4. The transforms

**Clarke (amplitude-invariant), 2 clk.** From two measured currents, with
$i_c$ implied by $i_a + i_b + i_c = 0$:

$$i_\alpha = i_a, \qquad i_\beta = \frac{i_a + 2 i_b}{\sqrt 3}$$

The intermediate $i_a + 2 i_b$ spans ±3.0, hence full-width arithmetic
and saturation only at the output. Amplitude-invariant scaling is used so
1.0 of d/q current equals 1.0 of phase current peak (same 1.25 A anchor).

**Park, 2 clk.** Rotation by $-\theta$ into the rotor frame:

$$i_d = i_\alpha \cos\theta + i_\beta \sin\theta, \qquad
  i_q = i_\beta \cos\theta - i_\alpha \sin\theta$$

Stage 1 registers the four DSP products; stage 2 sums in 33 bits, rounds
(>> 15) and saturates.

**Inverse Park, 2 clk.** Rotation by $+\theta$ back to the stator frame:

$$v_\alpha = v_d \cos\theta - v_q \sin\theta, \qquad
  v_\beta  = v_d \sin\theta + v_q \cos\theta$$

Round-trip accuracy (clarke → park → inv_park → TB inverse clarke) is
≤ 6 LSB for any in-range vector (`tb_transform_chain`).

## 5. The PI controllers (`pi_controller.sv`)

One instance per axis, $i_d^{ref} = 0$, $i_q^{ref}$ from UART. Difference
equations, bit-exact as documented (3-clk pipeline: error → gain
multiplies → sum/update):

$$e_k = \text{sat}(sp - fb)$$
$$u_k = \text{sat}\!\left(\text{round}_{12}(k_p e_k + I_k)\right)$$
$$I_{k+1} = \text{clamp}_{\pm 1.0}\!\left(I_k + k_i e_k + (applied_{k-1} - u_{k-1}) \cdot 2^{12}\right)$$

**Anti-windup by back-calculation**: `applied` is the *post-limiter*
value of the previous output (wired from `foc_core`'s vector limiter).
When nothing clamps, $applied = u$ and the correction term is zero; when
the limiter cuts the output, the integrator is pulled back by exactly the
cut amount, so it tracks the achievable output instead of winding up.
This is what lets the loop desaturate instantly (verified by the
ω-ramp-down windup scenario in `tb_foc_top`: no overshoot, no OCP trip
when a long-saturated loop regains authority).

### Tuning math

Plant per axis, in normalized units (1.0 V-unit = 24 V, 1.0 A-unit =
1.25 A): $G(s) = \frac{1}{R_s + sL_s} \cdot \frac{24}{1.25}$
A-unit/V-unit, with per-phase $R_s$ = 1.58 Ω, $L_s$ = 127 µH. The loop
also contains exactly **one PWM period of transport delay** ($T_s$ =
12.5 µs, §8) plus the ZOH half-period.

Design target (same as the STM32 reference): crossover
$\omega_c = 2\pi \cdot 1000$ rad/s with the PI zero on the plant pole:

$$k_p^{phys} = \omega_c L_s = 6283 \cdot 127\,\mu\text{H} = 0.798\ \text{V/A}, \qquad
  k_i^{phys} = \omega_c R_s = 9927\ \text{V/(A·s)}$$

Converting to the FPGA normalization ($\times\, 1.25/24$), folding $T_s$
into $k_i$, and scaling to Q4.12 ($\times\, 4096$):

$$k_p = 0.798 \cdot \tfrac{1.25}{24} \cdot 4096 = 170, \qquad
  k_i = 9927 \cdot 12.5\,\mu\text{s} \cdot \tfrac{1.25}{24} \cdot 4096 = 26$$

These are the reset defaults in `cmd_telemetry.sv`.

- Delay phase lag at crossover: $\omega_c \cdot 1.5 T_s \approx 0.12$ rad
  ≈ 7°, leaving ≈ 80° phase margin — comfortable.
- $k_i/k_p = 0.153$ per sample $\approx T_s/\tau_e = 0.156$: the PI zero
  **cancels the plant pole**, so the closed loop is approximately first
  order with $\omega_c$ bandwidth. Measured closed-loop 90 % rise in
  `tb_foc_top` is ≈ 30 PWM periods ≈ 366 µs $= \ln(10)/\omega_c$.
- The motor measured warm and with leads on the STM32 bench was
  $R_s \approx 2.83$ Ω; the matching integral gain is $k_i \approx 47$.
  The cold-datasheet 26 is baked as default; trim at the bench with the
  runtime `ki` command.

Gains are host-tunable at runtime (UART `kp`/`ki`) using the same Q4.12
scale.

**History note**: v1 shipped $k_p$ = 850 / $k_i$ = 130, derived against
the phase-to-phase R/L — against the true per-phase plant that is a
≈ 5 kHz crossover with only ≈ 34° of delay-induced margin loss, and the
closed-loop TB "verified" it against a plant built from the same
phase-to-phase values. Both were corrected together when aligning with
the STM32 reference.

The PI state is held cleared while the loop is disabled (`clr` = `!en`
in `foc_core`): re-enabling starts from zero integrators, the STM32
`foc_enable()` semantics. Enabling into a spinning rotor therefore
rushes current against the unopposed back-EMF — enable at standstill, as
the bring-up procedure does.

## 6. Output limiting and SVPWM

### d/q vector limiter (in `foc_core`)

The inverter's linear voltage range with min/max injection is a circle of
radius $V_{bus}/\sqrt 3$ (q15: `VMAX_Q15` = 18918 ≈ 0.577). The raw PI
outputs are clamped to that circle with **d priority** — $v_d$ keeps its
(range-clamped) value, $v_q$ gets the remainder:

$$v_q \leftarrow \text{sign}(v_q) \cdot \min\!\left(|v_q|, \sqrt{V_{max}^2 - v_d^2}\right)$$

d priority is correct for a torque loop: $v_d$ maintains field
orientation ($i_d = 0$), which is the prerequisite for $v_q$ producing
torque at all.

Two implementation notes, both learned the hard way:

- **Sequential isqrt.** A combinational 16-step non-restoring square root
  is ~77 logic levels (−50 ns slack). The limiter instead runs one isqrt
  iteration per clock: input registration → squares ($V_{max}^2 - v_d^2$)
  → 16 serial iterations → finish, ~20 clks total — noise against the
  1250-clk period.
- **`VD_CAP_Q15` (= 0.35 · VMAX ≈ 2.3 V).** The d axis physically needs
  only $|R i_d + \omega L i_q| < 1$ V for this motor. Without a cap, a
  *saturated q axis* lets the d integrator drift, and — because the
  limiter gives vd priority — the drift eats the whole circle:
  $v_q \rightarrow 0$, a stable wrong-way attractor observed in
  `tb_foc_top`. Capping closed-loop $v_d$ at 3× its physical requirement
  removes the attractor while leaving full orientation authority.

The post-limiter $(v_d, v_q)$ are fed back as `applied` to both PIs
(§5). A reference combinational version (`foc_pkg::dq_limit`) exists for
the TB golden model (`tb_dq_limit`).

### SVPWM by min/max injection (`svpwm.sv`, 3 clk)

Project $(v_\alpha, v_\beta)$ onto the three phase axes and inject the
zero-sequence offset that centers the envelope:

$$v_{a,b,c} = \left\{v_\alpha,\ -\tfrac{v_\alpha}{2} \pm \tfrac{\sqrt 3}{2} v_\beta\right\},
\qquad v_{off} = -\frac{\max(v) + \min(v)}{2}$$
$$d_x = 0.5 + (v_x + v_{off}) \quad \text{clamped to } 0.5 \pm \tfrac{MAX\_MOD}{2}$$

Zero-sequence injection is invisible line-to-line (the motor has no
neutral connection) but extends the linear range from $m = \pi/4$
(sine PWM) to $m = 1$ at $|v| = 1/\sqrt 3$, and is equivalent to
conventional space-vector modulation.

`MAX_MOD` = 0.87 is the **sampling guarantee**: duties never leave
$[0.065, 0.935]$, so every phase's low-side FET conducts ≥ 812 ns per
period around the counter peak — the XADC acquisition + settling budget
(see `docs/pwm.md` §1, §6). The per-phase clamp is a backstop; the d/q
limiter should keep vectors inside, and `sat` flags any clamping into
telemetry.

Pipeline: stage 1 registers the two DSP products (phase projection),
stage 2 the min/max + offset, stage 3 injects and clamps. Stages 2–3 run
on an 18/19-bit datapath ($|v_{phase}| < 2.0$ in Q3.15) — narrowing these
carry chains was the final timing-closure step at 100 MHz.

## 7. Sensing path (summary)

`xadc_iface` triggers the XADC (raw primitive, simultaneous-sampling mode
on the fixed VAUX0/VAUX8 pair) at `cnt_peak`, reads phases A and B, and
sequences a Vbus read. `current_offset_cal` averages 64 samples at
50/50/50 duty (gates *enabled*: realistic switching common mode) to
capture the CSA mid-bias offsets, subtracts them thereafter, and
reconstructs $i_c = \text{sat}(-i_a - i_b)$. Sampling at the ripple
midpoint (§ pwm.md) means the sampled value is the period-average
current, so the loop never sees the 0.30 A p-p ripple.

## 8. `foc_core` — the scheduler

One pass through the chain per current sample. Cycle budget after
`sample_valid` (which the XADC path fires once per PWM period):

```
t0   sample_valid: sincos_lut + clarke start (parallel)
t2   clarke done;  t3 sincos done -> park strobed
t5   park done -> both PIs strobed          (id_ref = 0, iq_ref)
t8   PI out_valid -> vector limiter         (reg, squares, 16x serial isqrt)
t27  limiter done -> inv_park strobed with limited vd/vq
t29  inv_park done -> svpwm strobed
t32  duties registered                      (~0.32 us total)
```

`pwm_gen` then double-buffers the duties at the *next* period boundary,
making the sample-to-applied-voltage delay **exactly one PWM period** —
a deliberate design constant (not "about 32 clocks") that the gain math
in §5 assumes. Everything is strobe/valid chained, so module latency
changes (as happened repeatedly during timing closure) never silently
skew the loop.

Modes and protection, all in `foc_core`:

- **open loop** (`ol_mode`): PIs frozen; host-supplied $(v_d, v_q)$ feed
  the limiter/inv_park directly, with θ from a host-rate accumulator in
  `foc_top` — used for V/f spin and Hall calibration (UART 0x08/0x09);
- **cal_active**: forces 50/50/50 duties during offset calibration;
- **OCP**: any sampled $|i| >$ `OCP_TRIP_Q15` (0.72 q15 = **0.9 A**)
  latches `ocp_trip`; only a rising edge of `en` re-arms. This is the
  *motor's* protection — the DRV8316's internal OCP (~16 A class)
  protects only the driver;
- `sat_any`: OR of every stage's saturation flag, exported in telemetry.

Telemetry format: `[0xAA][0x10][10][id, iq, θ, ω (int16 LE each)][fault_flags][status_flags][CHK]`,
every 5 ms. Bus voltage is not streamed (no Vbus sensing channel).

The safe-state gate (in `foc_top`, combinational between `pwm_gen` and
the pins):

```
oe = (enable | cal_busy) & nFAULT_sync & ~ocp_trip & ~wd_timeout & cfg_done
```

with `DRVOFF = ~oe` in parallel — an injected nFAULT kills all six gates
within one clock (asserted in `tb_foc_top`).

## 9. Verification map

| Property | Testbench |
|---|---|
| helper math, dq_limit golden model | `tb_foc_pkg`, `tb_dq_limit` |
| sin/cos < 1 LSB, all 2^16 angles | `tb_sincos_lut` |
| transforms vs. golden vectors + round trip | `tb_clarke/park/inv_park`, `tb_transform_chain` |
| PI vs. bit-exact model, windup release | `tb_pi_controller` |
| SVPWM duty range / centering / line-line / sat | `tb_svpwm` |
| scheduler dataflow, latency, OCP, cal | `tb_foc_core` |
| closed loop on the RL+BEMF plant: iq step tracking, id→0, windup desaturation without OCP, OCP trip + re-arm, watchdog ramp, 1-clk nFAULT kill | `tb_foc_top` (plant: `sim/bldc_plant.sv`) |

The plant model integrates $L \frac{di}{dt} = v - Ri - e$ per PWM period
with back-EMF $e_q = \omega K_e$ ($e_d = 0$), consistent with the
transform conventions above — so a closed-loop sign error anywhere in the
chain shows up as immediate divergence, not a subtle gain change.
